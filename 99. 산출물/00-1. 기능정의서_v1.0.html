기능 상세

기능동작비즈니스 규칙
개별 추가이름 입력 → classroom_students 레코드 생성같은 학급 내 동일 이름 불가
CSV 일괄 등록CSV 업로드 (이름 한 줄에 하나) → 일괄 생성중복 이름 건은 스킵 + 결과 리포트
이름 수정기존 이름 변경기존 시험 기록은 자동 연결 유지 (student_id 기반)
삭제소프트 삭제기존 시험 기록 보존. 응시 중 시험이 있으면 삭제 불가
자기 등록 관리학생이 스스로 입력한 이름에 "미확인" 뱃지 표시교사가 확인 클릭 → 뱃지 제거. 확인 전에도 응시 가능
동명이인: 같은 학급 내 동일 이름은 시스템이 차단합니다. 실제 동명이인이 있으면 교사가 구분자를 붙여 등록합니다 (예: "김민준(1번)", "김민준(2번)").
CSV 형식: UTF-8 인코딩. 헤더 없이 이름만 한 줄에 하나. 첫 행이 "이름" 또는 "name"이면 헤더로 인식하여 건너뜁니다.

L1 파이프라인 (문항 단위)

단계에이전트처리사용자 개입
PARSING#1 파싱팀HWP에서 텍스트, 수식(SCRIPT), 이미지(BINDATA), 표 추출없음

META#3 메타팀발문/선지/정답 구조화, 메타 태깅(단원·난이도·유형), 수식 LaTeX 변환없음
META_REVIEW#4 메타검수팀구조화 결과 검증. 불합격 시 META로 반려없음
PRODUCTION#5 제작팀정/오답 확정, AI 풀이 생성, 서술형 루브릭 생성, 렌더링 데이터 조립없음

HUMAN_CONFIRM—경로B(AI 도출 정답)만. 관리자 확인 대기관리자 (A-01)
DATA#7 데이터팀xAPI 설정 (MVP 스킵)없음
DATA_REVIEW#8 데이터검수팀xAPI 검증 (MVP 스킵)없음
EMBEDDING임베딩 모듈text-embedding-3-small로 벡터 생성, HNSW 인덱스 등록없음
L1_COMPLETED—완료. 문항 DB에 등록 완료. 시험지 구성(T-06)에서 사용 가능없음
검수 기준: 100점 만점, 85점 이상 합격. 단일 항목 배점의 50% 미만이면 총점 무관 자동 반려. 3회 연속 반려 → HUMAN_REVIEW(A-02).

유형별 채점 로직

유형채점 방식정규화정확도
객관식 (단일)정답 번호 == 학생 답 번호—100%
객관식 (복수, any)학생 선택 중 1개라도 정답에 포함 → 정답—100%
객관식 (복수, all)학생 선택 == 정답 집합 (순서 무관, 완전 일치)—100%
단답형허용 정답 목록과 비교공백 제거, 분수 정규화(½→0.5), 단위 무시 옵션99%+
서술형AI가 루브릭 기반 채점. 0~만점 사이 점수 부여 + confidence—참고 (교사 확인 권장)
서술형 채점: confidence: HIGH(루브릭 전 항목 명확 판단)/MEDIUM/LOW(핵심 논거 판단 불확실). LOW 문항은 T-11에서 교사 확인 강조.
채점 흐름: 제출 → 채점팀(#13) → 채점검수팀(#14, 85점 합격) → GRADE_COMPLETED → 교사 알림(T-15) + 학생 결과 공개(S-07).
