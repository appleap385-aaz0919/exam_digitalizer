01

L2 파이프라인 2단계 분리v1.3 핵심

시험지 "콘텐츠"와 "배포"를 분리합니다. 콘텐츠는 한 번 확정, 배포는 학급마다 반복.

L2-A: 시험지 콘텐츠 파이프라인 (exam_id 단위)

EXAM_COMPOSE→ EXAM_REVIEW→ EXAM_CONFIRMED
스테이지에이전트처리 내용
EXAM_COMPOSE#9 시험지 구성팀문항 선별, 밸런싱, 순서 배치, 자동 배점 할당
EXAM_REVIEW#10 시험지 검수팀조건 부합, 균형, 중복, 렌더링 검증
EXAM_CONFIRMED—교사 확정 완료. 콘텐츠 불변. 이후 여러 학급에 반복 배포 가능
EXAM_CONFIRMED 이후 콘텐츠 수정 불가. 문항 교체·배점 변경은 EXAM_COMPOSE로 돌아가야 합니다. 새 시험지를 만드는 것도 가능.

L2-B: 학급 배포 파이프라인 (classroom_exam_id 단위)

DEPLOY_REQUESTED→ HWP_GENERATING→ HWP_REVIEW→ DEPLOY_READY
DEPLOY_READY→ SCHEDULED→ ACTIVE→ CLOSED
스테이지에이전트처리 내용
DEPLOY_REQUESTED오케스트레이터교사가 학급에 시험 배포 요청 (opens_at/closes_at 설정)
HWP_GENERATING#11 서비스팀학급별 HWP 생성 (HWPML 템플릿 + 학급 전용 QR 삽입)
HWP_REVIEW#12 서비스검수팀HWP 무결성, QR 접근, 페이지 레이아웃, 원본 일치
DEPLOY_READY—HWP 다운로드 가능, 학생 접속 대기
SCHEDULED오케스트레이터opens_at 이전, 대기 상태
ACTIVE오케스트레이터opens_at 도래 → 학생 응시 가능
CLOSED오케스트레이터closes_at 도래 또는 수동 종료 → 미제출 자동 처리

L2-B 반려 흐름

// HWP_REVIEW에서 반려 시 HWP_REVIEW → REJECT → HWP_GENERATING (서비스팀이 재생성) // 콘텐츠 자체 문제 발견 시 (문항 오류 등) // L2-B에서 해결 불가 → L2-A의 EXAM_COMPOSE로 에스컬레이션 // 해당 classroom_exam은 DEPLOY_CANCELLED 상태로 전환

DB 변경사항v1.3

A. exams 테이블 변경

-- v1.3: hwp_file_path 제거 (학급별로 이동) -- exams는 콘텐츠만 담당 ALTER TABLE exams DROP COLUMN hwp_file_path; ALTER TABLE exams DROP COLUMN distribution_code; -- 학급 코드로 대체 ALTER TABLE exams DROP COLUMN qr_code_path; -- 학급별로 이동 -- status 값 변경 -- 기존: EXAM_REVIEW → ... → ACTIVE -- v1.3: EXAM_COMPOSE → EXAM_REVIEW → EXAM_CONFIRMED

B. classroom_exams 테이블 확장

컬럼타입제약v1.3 변경
idSERIALPK유지
classroom_idUUIDFK→classrooms유지
exam_idVARCHAR(20)FK→exams유지
opens_atTIMESTAMPTZ유지
closes_atTIMESTAMPTZ유지
hwp_file_pathVARCHAR(500)exams에서 이동
exam_qr_pathVARCHAR(500)유지 (학급+시험 인코딩 QR)
statusVARCHAR(20)확장: DEPLOY_REQUESTED → HWP_GENERATING → HWP_REVIEW → DEPLOY_READY → SCHEDULED → ACTIVE → CLOSED → DEPLOY_CANCELLED
created_atTIMESTAMPTZ유지

C. submission_answers.value 타입 확장 (복수정답)

-- value 컬럼은 TEXT 유지 (JSON 직렬화) -- 단일 선택: "3" -- 복수 선택: "[2, 4]" (JSON array 문자열) -- 단답형: "3cm" -- 서술형: "x=3이므로..." -- 미응답: null -- answer_type 확장 -- 기존: choice / short_answer / descriptive -- v1.3 추가: choice_multiple (복수 선택 객관식)

D. pipeline_states 변경

-- pipeline_level 확장 -- 기존: L1 / L2 -- v1.3: L1 / L2A / L2B -- L2A: ref_id = exam_id -- L2B: ref_id = classroom_exam_id (integer → varchar 변환 필요)

def grade_choice(student_value, answer_correct): correct_list = answer_correct["correct"] is_multiple = answer_correct.get("is_multiple", False) scoring_mode = answer_correct.get("scoring_mode", "any") if not is_multiple: # 단일 정답: value는 정수 return student_value == correct_list[0] else: # 복수 정답: value는 리스트 [2, 4] student_set = set(student_value) if isinstance(student_value, list) else {student_value} correct_set = set(correct_list) if scoring_mode == "any": # 정답 중 하나라도 선택하면 정답 return bool(student_set & correct_set) elif scoring_mode == "all": # 정답 전부를 정확히 선택해야 정답 return student_set == correct_set

Redis 보완v1.3

TTL 정책 명확화

# 세션 키 TTL 정책 (v1.3 최종) session:{sid}:info → TTL 없음. 제출/만료 후 cleanup으로 삭제 session:{sid}:answers → TTL 없음. 제출 시 DB 저장 후 cleanup으로 삭제 session:{sid}:heartbeat → TTL 60초 (자동 갱신, 미갱신 시 연결 끊김 감지) exam_expiry:{sid} → TTL = 시험시간(초). 만료 시 keyspace notification → 자동 제출 student_token:{token} → TTL 24시간 exam:{exam_id}:active → TTL 없음. CLOSED 시 cleanup exam:{exam_id}:progress → TTL 없음. CLOSED 시 cleanup

세션 cleanup 절차

async def cleanup_session(session_id, exam_id): # 1. Redis 답안 → DB 저장 (이미 제출된 경우 스킵) answers = await redis.hgetall(f"session:{session_id}:answers") if answers and not already_submitted(session_id): await save_to_db(session_id, answers, auto_submitted=True) # 2. Redis 키 삭제 await redis.delete( f"session:{session_id}:info", f"session:{session_id}:answers", f"session:{session_id}:heartbeat" ) # 3. 활성 세션 목록에서 제거 await redis.srem(f"exam:{exam_id}:active", session_id)

classroom_exam CLOSED 시 일괄 처리

async def close_classroom_exam(classroom_exam_id, exam_id): # 1. 활성 세션 전체 조회 active_sessions = await redis.smembers(f"exam:{exam_id}:active") # 2. 미제출 세션 자동 제출 for sid in active_sessions: await cleanup_session(sid, exam_id) # 3. classroom_exams.status = 'CLOSED' await db.update_status(classroom_exam_id, "CLOSED") # 4. 미응시 학생 기록 (submission 없는 학생) all_students = await db.get_classroom_students(classroom_exam_id) submitted_students = await db.get_submitted_students(classroom_exam_id) no_show = all_students - submitted_students for s in no_show: await db.create_submission(s, status="NO_SHOW")
